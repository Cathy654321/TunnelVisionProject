# Tunnel Vision Research Project

## Overview

Online social networks have reshaped public discourse by enabling large-scale, user-driven discussions on societal issues. However, such discussions often exhibit a progressive narrowing of attention, whereby collective focus converges on a limited subset of topic aspects while alternative perspectives are increasingly overlooked. We refer to this phenomenon as **tunnel vision**.

Unlike echo chambers, filter bubbles, or broader forms of ideological polarization, tunnel vision manifests at the **aspect level** — a loss of *topical diversity* over the stream of (aspect, sentiment) pairs, rather than a convergence of opinions. It is quantified with the **ASPE / CASE** diversity metrics implemented in `analysis_analyzer.py`.

This project formally defines tunnel vision, proposes a suite of mitigation strategies, and evaluates them on **two complementary tracks**:

1. **Controlled LLM-simulated discussion networks** — synthetic networks in which user behaviour is driven by Bayesian updating over aspect–sentiment choices, enabling systematic and ablatable evaluation of the mitigation strategies.
2. **Real-world Reddit discussions** (`reddit_exp5/`) — focused threads from the public Pushshift archive, annotated with genuine **aspect-based sentiment analysis (ABSA)**, used to confirm that aspect-level tunnel vision occurs in the wild and that the model reproduces its formation and mitigation.

All datasets, source code for the proposed algorithms, and complete experimental results and visualisations are publicly released to facilitate **reproducibility** and to enable other researchers to **learn from, extend, and reuse** our work.

## Part A — Simulated Discussion Networks

### 1. Snapshots (LLM-Driven Conversation Generation)

Snapshots are generated through **LLM-driven simulations**.

- An OpenAI API key is required for this step (see *Environment and Setup*).
- Generated conversations are saved as `.pkl` snapshots in the `snapshots/` directory.
- A total of **nine** simulated network/interaction settings are provided; **three representative datasets** are used in the reported experiments and the remaining six support extended studies.

### 2. Simulations (User Behaviour Modelling)

All simulations build on the generated snapshots.

- This stage performs **user behavioural simulation** guided by **Bayesian updating** over aspect–sentiment choices.
- The process is computationally efficient; saving outputs is optional and, when enabled, results are stored in `simulations/`.

### 3. Algorithms (Tunnel Vision Mitigation)

The mitigation algorithms are primarily **greedy-based** and implemented in `algorithm_collections.py`.

- Algorithms are injected into the network by setting the corresponding **algorithm flag** for each user.
- During comment-posting behaviour, the selected algorithm **augments the user's information view**, influencing aspect–sentiment exposure.

### 4. Experiments

Core experimental results are generated by `Experiments_main.py`.

- Per-configuration results are written to `./results_case/`.
- `utils_consolidate_csvs.py` consolidates them into `consolidated_results_case.csv` (further summaries in `exp_results/`).

### 5. Visualisations

- `Experiment_1.py`–`Experiment_4.py` (plus `Experiment_3_table.py` and `regenerate_figures.py`) generate all figures from the consolidated CSV into `Figures/`.

## Part B — Real-World Reddit Validation (`reddit_exp5/`)

A self-contained module that validates the framework on **three real Reddit debate threads** (Pushshift, April 2019), measuring aspect-level tunnel vision with genuine **ABSA** rather than topic modeling.

- Aspect–sentiment labels are produced by **gpt-5-mini** (ACSA against a frozen per-thread aspect schema) and **content-hash cached**, so the pipeline is offline, free, and reproducible after the first pass.
- The same Multifaceted Engagement Model + mitigation strategies are seeded from each thread's early phase to study formation and mitigation on real data.

## Code Structure

### Core Entities / Models

- `Event.py`, `SNTUser.py`, `Comment.py`, `SNTPlatform.py`


### Network Service

- `service_network.py`: create, load, and inspect snapshots and simulations.

### Analysis & Utilities

- `analysis_analyzer.py`: ASPE / CASE aspect-level diversity metrics.
- `utils_*.py`: helpers for data management, LLM access, sentiment analysis, and CSV consolidation.

### Real-Data Module

- `reddit_exp5/`: the real-world Reddit validation (Part B), with its own data, results, figures, and README.

## Environment and Setup

- **Python:** 3.14
- **Dependencies:** `requirements.txt`
- **Secrets:** rename `MY.env` to `.env` and fill in:
  - `OPENAI_API_KEY` — required for snapshot generation (Part A) and ABSA annotation (Part B).
  - `HF_TOKEN` — *optional*, for downloading Hugging Face ABSA models.

  `.env` is gitignored and never committed.

## Reproducibility

- All datasets, algorithms, and scripts are included.
- Part A snapshot regeneration requires an OpenAI API key; the simulation, experiment, and visualisation stages are deterministic given a snapshot.
- Part B (`reddit_exp5/`) reproduces every reported figure and table **offline** from the committed, content-hash-cached ABSA labels — no API key needed.
